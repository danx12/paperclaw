Findings

01 AGENTS.md / CLAUDE.md 💀 Broken
Neither AGENTS.md nor CLAUDE.md exists at the repo root. The only agent-adjacent file is .claude/skills/python-coding.md, a generic Python style skill — not project context.
Fix: create AGENTS.md with four sections: install (pixi install), test (pixi run test), self-test (pixi run demo → paperclaw …), and the Protocol seams in src/paperclaw/protocols.py the agent should respect.
02 Strict types / compiler 🩹 Patchy
pyproject.toml has no [tool.mypy] or [tool.pyright] block. Ruff ANN is selected (pyproject.toml:30) so missing annotations fail lint, but no static type checker is wired into pre-commit or CI.
Fix: add mypy to pixi.toml dev deps, [tool.mypy] strict = true in pyproject, and a pixi run typecheck task wired into .pre-commit-config.yaml.
03 Strict linter / formatter 🚀 Ship-Ready
pyproject.toml:30 selects E, W, F, I, UP, B, SIM, ANN — broad, not the starter set. Black is wired alongside. Both run in .pre-commit-config.yaml, and CI runs prek run --all-files --show-diff-on-failure so config is actually exercised.
04 Schema validation at boundaries 🚀 Ship-Ready
src/paperclaw/schemas.py defines RawDocument, ClassifiedDocument, LibraryDocument as frozen pydantic BaseModels with Field(ge=0.0, le=1.0), max_length=3, and a @model_validator that enforces .md sidecar suffix. Claude responses parse through the same schemas — the network boundary is typed.
05 Business logic separated from I/O 🚀 Ship-Ready
src/paperclaw/protocols.py declares Extractor / Classifier / Storer as typing.Protocols. pipeline.py is pure orchestration over those seams. _naming.py is pure functions. tests/test_pipeline.py exercises the full flow with in-memory fakes — no network or filesystem mocks.
06 One-command bring-up 🚀 Ship-Ready
pixi install (README:21) followed by uniform verbs pixi run test | lint | fmt | demo (pixi.toml:22-26). Single-package layout — no per-folder special cases for the agent to discover.
07 Pre-commit feedback loop 👍 Solid
.pre-commit-config.yaml runs ruff (autofix) + black + pytest (non-integration) + a custom .env file blocker. CI runs prek run --all-files so the hooks are genuinely exercised. Missing: gitleaks / detect-secrets — the .env filename pattern is the only secret guard.
08 Dead-code guardrail 🩹 Patchy
Ruff F (pyproject.toml:30) catches unused imports and unused local variables, but nothing catches unused functions, classes, or modules. No vulture, no deadcode, no RUF/PLR rules.
Fix: add vulture to dev deps and a pixi run lint step that runs vulture src/ --min-confidence 80, or add "RUF" to tool.ruff.lint.select.
09 Logs reachable from terminal 👍 Solid
logging.basicConfig(level=WARNING, …) at cli.py:14 plus typer.echo / typer.secho for status lines. Everything streams to stdout/stderr. No dedicated pixi run logs shortcut, but nothing is buried in a GUI.
10 Docs stay in sync with code 💀 Broken
README.md and docs/DESIGN.md exist, but nothing flags code-only changes. No pre-commit hook, no CI rule, no generated reference. Agent can edit schemas.py and the README will silently drift.
Fix: add a local prek hook that fails when src/** changes in a commit without a touch to README.md or docs/**, with a named bypass (PAPERCLAW_DOCS_OK=1).
11 Agent self-test end-to-end 👍 Solid
pixi run demo (pixi.toml:26) scaffolds /tmp/paperclaw-demo with bundled PDFs and prints the exact paperclaw … command. CLI emits stored / unsorted status to stdout and writes inspectable .md sidecars an agent can grep. Discoverable from README, not yet from AGENTS.md.
12 Agentic review panel 💀 Broken
No /review slash command, no REVIEW.md, no script that spawns parallel specialist reviewers. CI is pre-commit + integration tests only. Human is the first reviewer of every diff.
Fix: add .claude/commands/review.md that fans out python / security / best-practices reviewers in parallel, plus a short REVIEW.md ignore list (style covered by ruff/black, theoretical risks in unchanged code) and a diff-size tier rule.
13 Friction proportional to blast radius 💀 Broken
No CODEOWNERS, no pre-push danger-zone hook, no named bypass. The two highest-blast files — src/paperclaw/_naming.py (canonical filename touches every stored doc; renames break idempotency) and src/paperclaw/classifier.py (Claude prompt) — have no extra friction.
Fix: add a local prek hook that fails when those paths change without PAPERCLAW_DANGER_OK=1 in the env, and print a one-liner about what must have been done first (regression test against bundled PDFs, downstream sidecar parity check).
14 Tooling tuned for the agent 🩹 Patchy
The no-env-files hook (.pre-commit-config.yaml:7) prints a clear remediation: "Refusing to commit .env files — store secrets in env vars or config.toml". The ruff / black / pytest hooks just exit non-zero with stock output — no "run pixi run fmt" hint, no accept-list for known-false-positives.
Fix: wrap each hook's entry in a small shell wrapper that, on failure, prints "fix with: pixi run fmt" / "pixi run lint --fix". Add a .gitleaksignore (or similar accept-list) once gitleaks is wired so signal stays clean.

🎁 Bonus finds

  • 🎁 pixi run demo bootstrap (pixi.toml:26) — one command resets /tmp/paperclaw-demo, copies the bundled test PDFs into a fresh inbox, and prints the exact paperclaw … command to run next. The agent can verify a change end-to-end without inventing fixtures.
  • 🎁 Protocol seams + frozen pydantic models (protocols.py, schemas.py) — Extractor / Classifier / Storer are pure typing.Protocols and every domain object is frozen pydantic with validators. The agent can swap a fake classifier or storer in tests without touching the network or filesystem, and contract violations fail loudly.
